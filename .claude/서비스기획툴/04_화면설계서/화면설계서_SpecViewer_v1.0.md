## 공통 레이아웃

### 뷰어 공통 — 3단 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  [☰]  Spec Viewer         [🔍 검색창]   [💬] [⚙]   │  ← 상단 툴바 (height: 56px)
├──────────────┬──────────────────────────────────────┤
│              │                                      │
│   좌측       │         메인 콘텐츠 영역              │
│   사이드바   │                                      │
│  (240px)     │                                      │
│              │                                      │
│  · 파일 목록 │                                      │
│  · 목차      │                                      │
│              │                                      │
└──────────────┴──────────────────────────────────────┘
```

| 영역 | 크기 | 설명 |
|---|---|---|
| 상단 툴바 | height 56px, 전체 너비 | 로고, 글로벌 검색, 코멘트 패널 토글, 설정 |
| 좌측 사이드바 | width 240px, 고정 | 파일 목록 + 선택 문서 목차 |
| 메인 콘텐츠 | 나머지 너비 전체 | 화면별 콘텐츠 |

**반응형 분기점**

| 브레이크포인트 | 사이드바 동작 |
|---|---|
| ≥ 1280px | 항상 표시 (고정) |
| 768px ~ 1279px | 기본 숨김, 햄버거 버튼으로 오버레이 토글 |
| < 768px | 미지원 (데스크톱 전용 안내 표시) |

### 화면 개요

| 항목 | 내용 |
|---|---|
| 화면 ID | PKG-001 |
| 화면명 | 패키저 — 파일 등록 |
| 진입 경로 | GitHub Pages URL 직접 접속 |
| 주요 기능 | MD·HTML 파일 등록, 유형 지정, 화면 ID 매핑 설정 후 빌드 실행 |

### 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  Spec Viewer Packager                    [?] 도움말 │  ← 헤더
├─────────────────────────────────────────────────────┤

│  ① 파일 등록                                        │
│  ┌───────────────────────────────────────────────┐  │
│  │                                               │  │

### 컴포넌트 상세

#### ① 파일 업로드 영역

| 속성 | 내용 |
|---|---|
| 지원 형식 | `.md`, `.html`, `.htm` |
| 업로드 방식 | 드래그앤드롭 + 파일 선택 다이얼로그 (다중 선택 가능) |
| 드래그 진입 시 | 영역 테두리 강조(파란색), 배경색 변경 |
| 미지원 파일 등록 시 | 인라인 에러 메시지 표시 "지원하지 않는 파일 형식입니다 (.md, .html만 가능)" |

#### ② 파일 목록

| 컬럼 | 내용 |
|---|---|
| 파일명 | 등록된 파일명 표시. 길면 말줄임(...) 처리 |
| 유형 | 드롭다운. 옵션: 화면설계서 / PRD / 요구사항정의서 / 스토리보드 / 프로토타입. 파일명 패턴으로 자동 선택 |
| 크기 | KB 단위 표시. 10MB 초과 시 빨간색 경고 |
| 삭제 | [×] 클릭 시 목록에서 즉시 제거 |

**파일명 자동 분류 규칙**

| 파일명 패턴 | 자동 분류 유형 |
|---|---|
| `*spec*`, `*설계*`, `*화면*` | 화면설계서 |
| `*prd*`, `*PRD*` | PRD |
| `*요구*`, `*requirement*` | 요구사항정의서 |
| `*storyboard*`, `*스토리*` | 스토리보드 |
| `*.html`, `*.htm` | 프로토타입 |
| 해당 없음 | 화면설계서 (기본값) |

#### ③ 매핑 설정

| 모드 | 동작 |
|---|---|
| 자동 매핑 | MD 헤딩의 SCR-NNN 패턴 + HTML 파일명 SCR-NNN.html 패턴으로 자동 매핑. 매핑 테이블 읽기 전용으로 표시 |
| 자동 + 수동 보완 | 자동 매핑 결과를 편집 가능하게 표시. 미매핑 항목에 드롭다운으로 수동 지정 가능 |

- 자동 매핑 성공 행: 배경 연초록
- 자동 매핑 실패(미매핑) 행: 배경 연노랑 + "매핑 필요" 뱃지

#### ④ 출력 설정 및 빌드

| 항목 | 내용 |
|---|---|
| 파일명 | 기본값: `SpecViewer_v{버전}_{YYYYMMDD}`. 직접 편집 가능 |
| 빌드 버튼 | 파일이 1개 이상 등록되어야 활성화. 미등록 시 비활성(회색) |
| 빌드 실행 | PKG-002로 화면 전환 |

### 유효성 검사 및 에러

| 조건 | 처리 |
|---|---|
| 파일 미등록 상태에서 빌드 버튼 클릭 | 버튼 비활성 상태 유지 (클릭 불가) |
| 10MB 초과 파일 | 등록은 허용, 파일명 옆에 ⚠️ 경고 표시 "파일 크기가 큽니다. 빌드 결과물이 50MB를 초과할 수 있습니다" |
| 동일 파일명 중복 등록 | 덮어쓰기 확인 다이얼로그 표시 |

## PKG-002 패키저 — 빌드 결과

### 화면 개요

| 항목 | 내용 |
|---|---|
| 화면 ID | PKG-002 |
| 화면명 | 패키저 — 빌드 결과 |

| 주요 기능 | 빌드 진행 상태 표시, 완료 후 Viewer HTML 다운로드 |

### 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  Spec Viewer Packager                    [?] 도움말 │
├─────────────────────────────────────────────────────┤
│                                                     │
│  빌드 진행 중... (완료 시 결과로 전환)               │
│  ┌───────────────────────────────────────────────┐  │
│  │  [████████████████░░░░]  75%                 │  │
│  │  ✅ MD 파일 파싱 완료 (3개)                   │  │
│  │  ✅ HTML 변환 완료                            │  │
│  │  ⏳ 프로토타입 임베딩 중... (SCR-003.html)   │  │
│  │  ○  검색 인덱스 생성                          │  │
│  │  ○  단일 파일 패키징                          │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ── 빌드 완료 시 아래로 전환 ──                     │
│                                                     │
│  ✅ 빌드 완료                                       │
│  ┌───────────────────────────────────────────────┐  │
│  │  📄 SpecViewer_v1.0_20260518.html             │  │
│  │     크기: 8.2MB   소요 시간: 4.3초            │  │
│  │                                               │  │
│  │  [⬇ 다운로드]    [🔗 GitHub releases/ 안내]  │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  빌드 요약                                          │
│  · 포함 문서: 3개 (화면설계서 1, PRD 1, 요구사항 1) │

│  · 매핑된 화면 ID: 12개 (미매핑: 2개 ⚠️)          │
│                                                     │
│  [← 다시 빌드]                                     │
└─────────────────────────────────────────────────────┘
```

### 컴포넌트 상세

#### 빌드 진행 상태

| 단계 | 표시 |
|---|---|
| 대기 | ○ 회색 텍스트 |
| 진행 중 | ⏳ + 파란색 텍스트 + 스피너 |
| 완료 | ✅ 초록색 텍스트 |
| 실패 | ❌ 빨간색 텍스트 + 에러 메시지 |

**빌드 단계 순서**
1. MD 파일 파싱
2. HTML 변환
3. 프로토타입 임베딩
4. 검색 인덱스 생성
5. 단일 파일 패키징

#### 빌드 완료 상태

| 컴포넌트 | 동작 |
|---|---|
| 다운로드 버튼 | 브라우저 기본 다운로드로 파일 저장 |
| GitHub releases/ 안내 | releases/ 폴더에 커밋하는 방법 안내 모달 표시 |
| 빌드 요약 | 포함 문서 수, 프로토타입 수, 매핑 현황 표시 |
| 미매핑 경고 | ⚠️ 클릭 시 미매핑 화면 ID 목록 표시 |

#### 에러 처리

| 에러 유형 | 처리 |
|---|---|
| 빌드 실패 (파일 파싱 오류) | 해당 단계에 ❌ 표시, 에러 메시지 + 원인 파일명 표시, [← 다시 빌드] 버튼 활성 |
| 결과 파일 50MB 초과 | 완료는 허용, 경고 배너 표시 "파일이 큽니다. 브라우저에 따라 로딩이 느릴 수 있습니다" |

### 컴포넌트 상세

#### 상단 툴바

| 컴포넌트 | 동작 |
|---|---|
| 햄버거 [☰] | 사이드바 토글 (1280px 미만에서만 동작) |
| 로고 "Spec Viewer" | 클릭 시 SCR-001(홈)으로 이동 |
| 글로벌 검색창 | 포커스 시 검색 드롭다운 표시. 키워드 입력 → SCR-005로 이동. 화면 ID(SCR-NNN) 입력 → 해당 섹션 직접 이동 |
| 코멘트 [💬] | SCR-006 코멘트 패널 슬라이드인 |
| 설정 [⚙] | 설정 드롭다운 (뷰어 정보, 즐겨찾기 초기화, 코멘트 초기화) |

#### 좌측 사이드바

| 컴포넌트 | 동작 |
|---|---|
| 파일 목록 | 등록된 문서를 유형별로 그룹화하여 표시. 클릭 시 SCR-002로 이동하며 해당 문서 로드 |
| 프로토타입 목록 | 등록된 HTML 파일 목록. 클릭 시 SCR-003으로 이동 |
| 활성 항목 | 현재 열람 중인 문서·프로토타입 좌측에 파란색 바(border-left) 표시 |

#### 즐겨찾기

| 속성 | 내용 |
|---|---|
| 저장 위치 | 브라우저 LocalStorage |
| 카드 표시 | 화면 ID + 화면명. 최대 6개까지 표시, 초과 시 "더보기" |
| 카드 클릭 | 해당 화면 ID의 문서 섹션으로 이동 |
| 즐겨찾기 없을 때 | "아직 즐겨찾기가 없습니다. 문서에서 ⭐을 눌러 추가해보세요." 안내 문구 |

#### 최근 열람

| 속성 | 내용 |
|---|---|
| 저장 위치 | 브라우저 LocalStorage |
| 표시 개수 | 최대 10개 |
| 항목 | 화면 ID 또는 문서명 + 섹션명 + 상대 시간 (방금 전 / 오늘 / N일 전) |
| 항목 클릭 | 해당 위치로 이동 |

## SCR-002 뷰어 — 문서 뷰어

### 화면 개요

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-002 |
| 화면명 | 뷰어 — 문서 뷰어 |

| 주요 기능 | MD 문서 렌더링, 목차 탐색, 화면 ID 앵커, 코멘트 작성 |

### 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  [☰]  Spec Viewer       [🔍 검색창]      [💬] [⚙]  │
├──────────────┬──────────────────────┬───────────────┤
│ 📁 파일 목록 │                      │  목차          │

## SCR-003 뷰어 — 프로토타입 뷰어

### 화면 개요

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-003 |
| 화면명 | 뷰어 — 프로토타입 뷰어 |

| 주요 기능 | HTML 프로토타입 iframe 실행, 디바이스 프리뷰 전환 |

### 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  [☰]  Spec Viewer       [🔍 검색창]      [💬] [⚙]  │
├──────────────┬──────────────────────────────────────┤
│ 📁 파일 목록 │ SCR-001.html  [문서 보기] [스플릿 뷰] │  ← 서브 툴바
│ ──────────── │ [📱Mobile] [📟Tablet] [🖥Desktop] [✏Custom] │
│ ▶ SCR-001    │ ┌──────────────────────────────────┐ │
│   SCR-002    │ │                                  │ │
│   SCR-003    │ │                                  │ │
│              │ │      iframe (프로토타입 실행)      │ │
│              │ │                                  │ │
│              │ │   (375 × 812 — Mobile 기준)      │ │
│              │ │                                  │ │
│              │ └──────────────────────────────────┘ │
│              │  현재: 375 × 812px  [새 탭에서 열기] │
└──────────────┴──────────────────────────────────────┘
```

### 컴포넌트 상세

#### 서브 툴바

| 컴포넌트 | 동작 |
|---|---|
| 파일명 표시 | 현재 열람 중인 프로토타입 파일명 |
| [문서 보기] | 매핑된 문서의 해당 화면 ID 섹션으로 이동 (SCR-002) |
| [스플릿 뷰] | SCR-004로 전환. 현재 프로토타입 + 매핑 문서 섹션 유지 |

#### 디바이스 프리뷰

| 프리셋 | 너비 × 높이 | 비고 |
|---|---|---|
| 📱 Mobile | 375 × 812 | iPhone 14 기준, 기본값 |
| 📟 Tablet | 768 × 1024 | iPad 기준 |
| 🖥 Desktop | 1440 × 900 | 기본 데스크톱 |
| ✏ Custom | 직접 입력 | 너비·높이 별도 입력 필드 표시 |

- 선택한 프리셋에 따라 iframe 크기 변경
- iframe 외부 영역은 체크무늬 배경으로 "화면 바깥" 시각적 구분
- 메인 영역 너비보다 iframe이 클 경우 가로 스크롤 표시

#### iframe 설정

| 속성 | 값 |
|---|---|
| sandbox | `allow-scripts allow-same-origin allow-forms` |
| 렌더링 방식 | srcdoc 우선, 실패 시 Blob URL 폴백 |
| 로딩 실패 시 | "프로토타입을 불러올 수 없습니다" 안내 + 원인 메시지 표시 |

### 컴포넌트 상세

#### 분할 패널

| 속성 | 내용 |
|---|---|
| 기본 비율 | 좌(문서) 50% : 우(프로토타입) 50% |

- SCR-002 문서 뷰어와 동일한 렌더링
- 목차 패널은 공간 부족으로 미표시 (사이드바 목차만 사용)

#### 우측 — 프로토타입 패널

- SCR-003 프로토타입 뷰어와 동일한 iframe + 디바이스 프리뷰 툴바
- 문서 패널의 화면 ID와 매핑된 프로토타입 자동 로드

#### 스플릿 뷰 종료

- 상단 툴바 [스플릿 뷰 ✕] 클릭 시 종료
- 종료 후 직전 화면(SCR-002 또는 SCR-003)으로 복귀
