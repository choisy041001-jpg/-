## 1. 문서 개요

### 1.1 프로젝트 개요

| 항목 | 내용 |
|---|---|
| **프로젝트명** | Spec Viewer — 화면설계서 통합 뷰어 툴 |
| **목적** | 서비스 기획자가 작성한 화면설계서(MD), PRD, 요구사항정의서, 스토리보드 및 HTML 프로토타입을 단일 HTML 파일로 패키징하여 팀원 누구나 브라우저에서 바로 열람 가능하도록 제공 |
| **실행 방식** | 단일 HTML 파일 — 파일을 내려받아 브라우저에서 열면 즉시 동작 (서버·설치 불필요) |
| **주요 사용자** | 서비스 기획자, UI/UX 디자이너, 개발자, QA 담당자 |
| **공유 방법** | HTML 파일을 메신저·이메일·드라이브 등으로 전달 |
| **문서 작성일** | 2026-05-18 |
| **버전** | v1.3 (로컬 실행 방식 반영) |

### 1.2 배경 및 필요성

서비스 기획자는 프로젝트 진행 과정에서 다양한 산출물을 생성하지만, 이를 통합적으로 탐색·열람할 수 있는 전용 도구가 없어 다음과 같은 문제가 발생한다.

- 산출물 파일이 분산 관리되어 최신본 파악이 어려움
- MD 문서와 HTML 프로토타입 간 연결 고리 부재로 컨텍스트 파악 불편
- 팀원 공유 시 여러 파일을 개별 전달해야 하는 비효율
- 화면 ID 기반 탐색 기능 부재로 특정 화면 검색에 시간 소요
- 서버 배포 없이 누구나 즉시 열람할 수 있는 방법 필요

→ **단일 HTML 파일 하나를 열기만 하면 모든 기획 산출물을 탐색할 수 있는 Spec Viewer로 위 문제를 해결한다.**

### 1.3 용어 정의

| 용어 | 정의 |
|---|---|
| **화면설계서** | 서비스 화면의 UI 구성, 기능, 플로우를 정의한 MD 형식의 문서 |
| **PRD** | Product Requirements Document. 제품 요구사항을 정리한 문서 |
| **스토리보드** | 화면 흐름과 인터랙션을 시각적으로 정리한 기획 문서 |
| **프로토타입** | 기능이 구현된 HTML 파일 형태의 시연 화면 |
| **화면 ID** | 각 화면을 식별하는 고유 식별자 (예: `SCR-001`) |
| **Spec Viewer** | 본 프로젝트에서 개발하는 단일 HTML 파일 형태의 화면설계서 통합 뷰어 |
| **단일 HTML** | 모든 리소스 (스타일, 스크립트, 문서 데이터)가 하나의 `.html` 파일에 인라인으로 포함된 형태 |

## 2. 이해관계자 및 사용자 분석

### 2.1 이해관계자

| 역할 | 주요 관심사 | 기대 효과 |
|---|---|---|
| **서비스 기획자** | 문서 등록·패키징 효율화, 프로토타입 연동 | 단일 HTML 하나로 산출물 일괄 공유 |
| **UI/UX 디자이너** | 설치 없이 화면 명세 즉시 조회 | 파일 열람만으로 기획 의도 빠른 파악 |
| **개발자** | 기능 명세·화면 ID 기반 검색 | 요구사항 오해 감소, 탐색 시간 단축 |
| **QA 담당자** | 테스트 시나리오 대비 화면 명세 확인 | 별도 환경 없이 화면 명세 즉시 확인 |

### 2.2 핵심 사용 시나리오

**AS-IS (현재 문제 상황)**

- 기획자는 MD 파일, HTML 파일을 별도 폴더에서 관리하며 팀원 요청 시마다 개별 파일 전달
- 팀원은 여러 파일을 각각 열어서 확인해야 하며 최신 버전 여부를 항상 확인해야 함
- 화면 ID를 알아도 해당 화면 명세를 찾으려면 수동으로 문서를 탐색해야 함

**TO-BE (목표 상황)**

- 기획자는 Spec Viewer 생성 도구에 MD·HTML 파일을 넣으면 단일 HTML 파일 하나가 생성됨
- 팀원은 HTML 파일 하나를 받아 브라우저에서 열기만 하면 모든 문서·프로토타입 즉시 열람 가능
- 화면 ID 입력 또는 클릭으로 해당 화면의 명세와 프로토타입을 동시에 확인

## 5. 제약사항 및 전제조건

### 5.1 제약사항

- 지원 파일 형식은 MD (`.md`), HTML (`.html`, `.htm`)으로 제한한다
- 단일 HTML 파일 특성상 서버 사이드 기능 (사용자 인증, 실시간 동기화, DB 저장 등)은 제공하지 않는다
- HTML 프로토타입 내 외부 네트워크 요청은 오프라인 환경 또는 브라우저 보안 정책에 따라 제한될 수 있다
- 즐겨찾기·최근 열람·코멘트 등 상태 데이터는 브라우저 LocalStorage에 저장되며 다른 기기에서는 공유되지 않는다
- 패키저 웹 UI는 GitHub Pages를 통해 배포되므로 접근 시 인터넷 연결이 필요하다 (생성된 Viewer HTML은 오프라인 동작)

### 5.2 전제조건

- 팀원은 최신 웹 브라우저가 설치된 환경에서 파일을 열람한다
- 소스코드 및 빌드 산출물은 GitHub 레포지터리에서 관리된다
- 패키저 웹 UI는 GitHub Pages (`docs/` 폴더)로 배포되며, 팀원은 해당 URL로 접근할 수 있다
- 빌드된 Viewer HTML 파일은 레포지터리 `releases/` 폴더에 커밋되어 팀원이 직접 다운로드한다
- 화면설계서 MD 파일 내 화면 ID는 일관된 형식 (예: `SCR-NNN`)으로 작성되어 있어야 자동 매핑이 가능하다
- HTML 프로토타입은 단일 HTML 파일이거나 관련 리소스가 인라인으로 포함된 형태이어야 한다
