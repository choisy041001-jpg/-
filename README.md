# 서비스 기획 문서 관리

## 문서 구조

| 폴더 | 설명 |
|------|------|
| `01_요구사항정의서` | 서비스 요구사항 정의 문서 |
| `02_PRD` | Product Requirements Document |
| `03_스토리보드` | 사용자 흐름 및 스토리보드 |
| `04_화면설계서` | 화면별 상세 설계 문서 |
| `05_IA` | Information Architecture |
| `06_플로우차트` | 서비스 플로우 다이어그램 |

## 브랜치 전략

```
main                    확정된 문서 (최종본)
feature/[기능명]-[문서]  작업 중인 문서
review/[스프린트명]      검토 요청 중인 문서
```

## 커밋 메시지 규칙

```
feat: 새 문서 추가
update: 기존 문서 수정
fix: 오류 수정
review: 검토 의견 반영
```

## 문서 상태 표기

모든 문서 상단에 상태를 표기합니다.

- `작성중` — 초안 작성 단계
- `검토중` — 팀 내부 검토 단계
- `확정` — 최종 승인 완료
